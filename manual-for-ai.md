# toulmin ‚Äî Manual for AI Agents

Toulmin argumentation-based rule engine for Go. Rules are Go functions. Engine builds defeats graph and computes verdict via h-Categoriser.

## How to Navigate

1. Read `codebook.yaml` ‚Äî project vocabulary (required/optional keys and allowed values)
2. `rg '//rule:warrant'` ‚Äî find warrant rules
3. `rg '//rule:defeater'` ‚Äî find defeater rules, check `defeats=` targets
4. Read `//rule:what` to understand each rule ‚Äî skip body if what is sufficient
5. Full read only the files you need, then work

---

## Core Concepts

### Rule

All rules share one signature:

```go
func(claim any, ground any) bool
```

A rule returns `true` when its condition holds. The engine doesn't look inside ‚Äî it only needs the bool result.

### Strength

| Strength | Effect |
|---|---|
| Strict | Rejects all incoming attack edges |
| Defeasible | Accepts incoming attack edges |
| Defeater | Outgoing attack edges only, no own verdict |

### Defeats Graph

Rules form a directed graph. Edges represent attacks:

- **Warrant**: node that can be attacked (primary rule)
- **Rebuttal**: node that attacks a warrant (has own conclusion)
- **Defeater**: node that attacks without own conclusion (blocks only)

The distinction is not in the type ‚Äî it's in the graph position (`Defeats` field).

### Verdict

h-Categoriser computes verdict on [-1, +1] scale:

```
raw(a) = w(a) / (1 + Œ£ raw(attackers))     [0, 1]
verdict(a) = 2 √ó raw(a) - 1                [-1, 1]
```

| Verdict | Meaning |
|---|---|
| +1.0 | Violation confirmed (warrant holds, no attacks) |
| 0.0 | Undecided (warrant fully neutralized) |
| -1.0 | Fully rebutted |
| > 0 | Warrant prevails |
| < 0 | Rebuttal prevails |

### Qualifier

`0.0‚Äì1.0` float. Initial weight for h-Categoriser. Default: `1.0`.

## Writing Rules

### Annotations

Write `//rule:` comments directly above the function declaration:

```go
//rule:warrant qualifier=1.0 strength=strict
//rule:backing "B√∂hm-Jacopini theorem"
//rule:what F1: one func per file
func CheckOneFileOneFunc(claim any, ground any) bool {
    return ground.(FileGround).FuncCount > 1
}

//rule:defeater defeats=CheckOneFileOneFunc
//rule:backing "test files conventionally group multiple test funcs"
//rule:what F5: test files allow multiple funcs
func TestFileException(claim any, ground any) bool {
    return strings.HasSuffix(claim.(string), "_test.go")
}
```

| Annotation | Required | Description |
|---|---|---|
| `//rule:warrant` | for warrants | Declares warrant. Params: `qualifier=`, `strength=` |
| `//rule:defeater` | for defeaters | Declares defeater. Params: `defeats=`, `qualifier=`, `strength=` |
| `//rule:backing` | yes | Why this rule exists. Quoted string |
| `//rule:what` | yes | One-line description |

### Strength values

- `strict` ‚Äî cannot be attacked
- `defeasible` ‚Äî can be attacked (default for warrant)
- `defeater` ‚Äî attacks only, no own verdict (implicit for `//rule:defeater`)

### Qualifier

Omit for default `1.0`. Override with `qualifier=0.85` for uncertain rules.

## Using the Engine (Library)

### Manual Registration

```go
eng := toulmin.NewEngine()

eng.Register(toulmin.RuleMeta{
    Name:      "CheckOneFileOneFunc",
    Qualifier: 1.0,
    Strength:  toulmin.Strict,
    Backing:   "B√∂hm-Jacopini theorem",
    What:      "F1: one func per file",
    Fn:        CheckOneFileOneFunc,
})

results := eng.Evaluate(claim, ground)
for _, r := range results {
    fmt.Printf("%s: verdict=%f\n", r.Name, r.Verdict)
}
```

### Generated Registration

Use `toulmin graph` to auto-generate `RegisterAll(eng)` from annotations:

```go
eng := toulmin.NewEngine()
RegisterAll(eng) // generated by toulmin graph
results := eng.Evaluate(claim, ground)
```

## Commands

```bash
toulmin graph <dir>                          # generate register_gen.go
toulmin graph <dir> --dry-run                # print to stdout
toulmin graph <dir> --output path/out.go     # custom output path
toulmin evaluate                             # run example evaluation
```

### toulmin graph

1. Scans Go source files in `<dir>` (skips `_test.go`, `_gen.go`)
2. Extracts `//rule:` annotations + function names via Go AST
3. Validates defeats references (unknown target ‚Üí error, exit 1)
4. Generates `register_gen.go` with `RegisterAll(eng *toulmin.Engine)`

## Evaluation Flow

```
1. Run all rules: func(claim, ground) ‚Üí bool
2. Collect activated rules (true only)
3. Build subgraph: activated nodes + defeats edges
   (strict nodes reject incoming edges)
4. h-Categoriser: raw(a) = w(a) / (1 + Œ£ raw(attackers))
   verdict(a) = 2 * raw(a) - 1
   Circular attack: maxDepth(100) returns 0.0
5. Final judgment: verdict > 0 ‚Üí violation, == 0 ‚Üí undecided, < 0 ‚Üí rebutted
```

## Verdict Examples

### Warrant only (no attacks)

```
W "one-func-per-file" w=1.0, attackers: none
raw = 1.0 / (1+0) = 1.0 ‚Üí verdict = +1.0 (violation confirmed)
```

### Warrant + defeater

```
W "one-func-per-file" w=1.0
  attacker: D "test-file-exception" w=1.0

raw(D) = 1.0 ‚Üí raw(W) = 1.0/(1+1.0) = 0.5 ‚Üí verdict(W) = 0.0 (undecided)
```

### Compensation (defense)

```
W "one-func-per-file" w=1.0
  attacker: D1 "test-file-exception" w=1.0
    attacker: D2 "test-helper-not-exception" w=1.0

raw(D2) = 1.0
raw(D1) = 1.0/(1+1.0) = 0.5
raw(W)  = 1.0/(1+0.5) = 0.667 ‚Üí verdict(W) = +0.33 (partially restored)
```

### Strict warrant (rejects attack)

```
W "one-func-per-file" w=1.0 strength=strict
  attacker: D ‚Äî edge rejected

verdict(W) = +1.0 (unchanged)
```

## Common Mistakes

| Mistake | Fix |
|---|---|
| Missing `//rule:what` | Always write what annotation |
| `defeats=` targets unknown rule | Check rule name exists. Run `toulmin graph` to validate |
| Qualifier 0.0 (rule has no weight) | Omit qualifier for default 1.0, or set explicitly |
| Rule func wrong signature | Must be `func(claim any, ground any) bool` |
| Editing `register_gen.go` manually | Re-run `toulmin graph` instead. File is generated |

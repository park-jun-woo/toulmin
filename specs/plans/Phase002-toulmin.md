# Phase 002: toulmin graph — 소스 스캔 + 등록 코드 생성

## 목표

`toulmin graph` 커맨드 구현.
Go 소스 디렉토리를 스캔하여 `//rule:` 어노테이션과 함수 시그니처를 추출하고,
defeats 그래프를 구성한 뒤 `RegisterAll(eng)` 함수를 Go 코드로 생성한다.

## 배경

Phase 001에서 엔진 코어는 완성되었으나, 규칙 등록이 수동이다.
소비자가 `eng.Register(RuleMeta{...})`를 일일이 작성해야 한다.
`toulmin graph`는 이 과정을 자동화한다.

## 범위

### 포함

1. **소스 스캐너**: Go AST로 소스 파일 순회, `//rule:` 주석 + 함수명 추출
2. **그래프 검증**: defeats 참조가 유효한지 (존재하지 않는 rule 공격 등) 검증
3. **코드 생성**: `RegisterAll(eng *toulmin.Engine)` 함수를 Go 파일로 출력
4. **CLI**: `toulmin graph <dir>` 커맨드

### 제외

- 런타임 동적 로딩 (Go는 컴파일 언어, 코드 생성이 정답)

## 입력 → 출력

### 입력: 소스 파일들

```go
//rule:warrant qualifier=1.0 strength=strict
//rule:backing "Böhm-Jacopini theorem"
//rule:what F1: one func per file
func CheckOneFileOneFunc(claim any, ground any) bool {
    return ground.(FileGround).FuncCount > 1
}

//rule:defeater defeats=CheckOneFileOneFunc
//rule:backing "test files conventionally group multiple test funcs"
//rule:what F5: test files allow multiple funcs
func TestFileException(claim any, ground any) bool {
    return strings.HasSuffix(claim.(string), "_test.go")
}
```

### 출력: 생성된 등록 코드

```go
// Code generated by toulmin. DO NOT EDIT.
package <소스 패키지명>

import "github.com/park-jun-woo/toulmin/pkg/toulmin"

func RegisterAll(eng *toulmin.Engine) {
    eng.Register(toulmin.RuleMeta{
        Name:      "CheckOneFileOneFunc",
        Qualifier: 1.0,
        Strength:  toulmin.Strict,
        Backing:   "Böhm-Jacopini theorem",
        What:      "F1: one func per file",
        Fn:        CheckOneFileOneFunc,
    })
    eng.Register(toulmin.RuleMeta{
        Name:      "TestFileException",
        Qualifier: 1.0,
        Strength:  toulmin.Defeater,
        Defeats:   []string{"CheckOneFileOneFunc"},
        Backing:   "test files conventionally group multiple test funcs",
        What:      "F5: test files allow multiple funcs",
        Fn:        TestFileException,
    })
}
```

## 단계

### Step 1: 소스 스캐너

- `go/ast`, `go/parser`로 Go 소스 파일 파싱
- 함수 선언(FuncDecl) 위의 주석 그룹에서 `//rule:` 접두사 필터링
- 함수명 + 어노테이션 → RuleMeta 리스트 반환 (Fn=nil)

### Step 2: 그래프 검증

- defeats 참조가 실제 존재하는 rule을 가리키는지 확인
- strict rule을 공격하려는 선언이 있으면 경고
- 누락된 참조 → 에러 출력, exit 1

### Step 3: 코드 생성

- RuleMeta 리스트 → Go 소스 코드 문자열 생성
- `// Code generated by toulmin. DO NOT EDIT.` 헤더
- package명은 스캔 대상 디렉토리의 패키지를 따름
- Fn 필드는 함수명을 그대로 참조 (같은 패키지 내 컴파일타임 바인딩)
- 출력 파일: `<dir>/register_gen.go`

### Step 4: CLI 통합

- `toulmin graph <dir>` — 디렉토리 스캔 → 검증 → 코드 생성
- `--output` 플래그로 출력 경로 지정 가능 (기본: `<dir>/register_gen.go`)
- `--dry-run` 플래그로 생성 결과 stdout 출력

## 검증 기준

1. `//rule:` 어노테이션이 있는 함수만 추출된다
2. 어노테이션이 없는 함수는 무시된다
3. defeats 참조가 존재하지 않는 rule을 가리키면 에러
4. 생성된 `register_gen.go`가 컴파일된다
5. 생성된 `RegisterAll`로 등록 후 `Evaluate` 결과가 수동 등록과 동일하다
